# Systematic Review Methodology

A systematic review follows a reproducible protocol to identify, evaluate, and synthesise all relevant studies on a specific question, minimising selection bias.

## Core Steps

1. **Define the question** using PICO (Population, Intervention, Comparison, Outcome) or equivalent framework
2. **Register the protocol** (PROSPERO, OSF) before searching
3. **Search systematically** across multiple databases with documented queries
4. **Screen** by title/abstract, then full text, with inclusion/exclusion criteria
5. **Extract data** into a structured form
6. **Assess quality** using domain-appropriate tools (e.g., Cochrane RoB, CASP, JBI)
7. **Synthesise** — narrative, meta-analysis, or thematic

## Common Pitfalls

- Publication bias — supplement database searches with grey literature, preprints, and hand-searching reference lists
- Language bias — restricting to English excludes valid evidence
- Narrative cherry-picking — use PRISMA flow diagrams to document every exclusion

## When Not to Use

- When the field is too nascent for sufficient primary studies — use a [[scoping-review-vs-systematic-review|scoping review]] instead
- When the question is too broad — narrow with specific outcomes or populations
