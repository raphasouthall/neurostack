# Retrieval Practice

Retrieval practice — actively pulling information from memory rather than passively reviewing it — is the single most effective study strategy identified by cognitive science, consistently outperforming re-reading, highlighting, and summarising.

## The Testing Effect

Retrieving a memory modifies and strengthens it more than re-exposure does (Roediger & Karpicke, 2006). Every successful retrieval makes the next retrieval easier and extends the memory's durability. Even failed retrieval attempts improve subsequent learning, provided feedback follows.

## How to Implement

- **Flashcards** — use [[spaced-repetition]] software (Anki, Mochi) to schedule retrieval at optimal intervals
- **Practice tests** — attempt past exams under timed conditions before reviewing answers
- **Blank page recall** — close your notes, write everything you remember about a topic, then compare
- **Teach it** — explain the concept aloud or to a study partner, using the [[feynman-technique]]
- **Cornell cue column** — cover the notes side of your [[cornell-note-method]] page and answer each cue

## When to Retrieve

- **Immediately after class** — write a summary from memory before checking notes
- **Before the next class** — spend 5 minutes retrieving the previous lecture's key points
- **At expanding intervals** — 1 day, 3 days, 7 days, 21 days (aligns with [[spaced-repetition]])

## Common Mistakes

- Confusing recognition with recall — re-reading feels easy because you recognise the material, but recognition is not retrieval
- Skipping feedback — retrieval without checking accuracy can reinforce errors
- Avoiding difficulty — the harder the retrieval, the stronger the learning; easy recall means the interval should increase
